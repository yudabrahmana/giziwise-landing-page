Jelajahi Dunia Nutrisi dengan GiziWise

We are C23-PS450 | Product-Based Bangkit Capstone Project Team

Abstract

The project aims to develop a mobile app for tracking the nutritional content of food. The app will use the latest nutritional data, consider regional variations, and focus on simplicity, accuracy, and ease of use. In the end, the app aims to empower users to make informed decisions, improve their health, and reduce healthcare costs.

With the rise of chronic diseases and an increasing interest in health and well-being, there is a growing need for a tool that accurately and easily tracks the nutritional content of food. The application can help users make informed decisions about their diets, potentially reducing the incidence of chronic diseases and their associated healthcare costs.

Machine Learning Part

Our machine learning will involve a Convolutional Neural Network algorithm that uses the TensorFlow library. It will be used to train a dataset to build a multiclass classifier for ingredients, such as vegetable and fruit recognition. Other than that, we will use transfer learning as well if it is possible.

For machine learning, we will use Jupyter Notebook, Google Colab, or some similar app as our tools. The TensorFlow library will definitely be used. Keras, Matplotlib, and Numpy will also be used as supporting libraries. The dataset most likely will be fetched from Kaggle. Moreover, to train the model, we will use references from StackOverflow, Github Repositories, and other similar projects. We will try to do transfer learning as well whenever possible.

Mobile Development Part

Our Mobile Development capstone project involves creating a Kotlin-based Android app using Android Studio as the IDE. We will use the Android Jetpack library for efficient development and focus on creating a user-friendly app that solves real-world problems. Figma will be used for UI design and prototyping.

In Mobile Development, we will use Figma, GitHub, and Android Studio as tools and IDE. We will also use the Android Jetpack and Retrofit as libraries. Android SDK and Google Play Services as the platform/API.

Cloud Computing Part

Our back end tech stack will be Nodejs, Typescript, Fastify, Sequelize, and Postgres. There will be authentication using Firebase Sign-in if visitors want to register for additional features such as search history, diet plan, and others. Features that use machine learning models will be served using Google Cloud Functions.

In Cloud Computing we will use Cloud SQL to store structured data, Kubernetes Engine to deploy the app, Cloud Functions for deploying the machine learning model on the cloud, Cloud Storage service to store and manage images of food items.
